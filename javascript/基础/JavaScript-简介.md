# JavaScript 简介

```nginx
# JavaScript 是一种以纯文本的形式提供和执行。它不需要特殊的准备或编译即可运行的脚本(程序)。

# 它可以运行在客户端上,也可以运行在服务端上,必要时会嵌入了 JavaScript 引擎(也称 JavaScript 虚拟机)。

# JavaScript 引擎工作原理
 1. 通过脚本引擎读取脚本。
 2. 脚本引擎会将脚本文件内容编译成机器语言。
 3. 已编译的机器语言程序块在运行环境里快速执行。

# JavaScript 可行行为
总的来说,浏览器中的 JavaScript 可以做与网页操作、用户交互和 Web 服务器相关的所有事情。实际上取决于运行环境的开放程度。

# JavaScript 不可行行为
 1. JavaScript 不能读、写、复制和执行硬盘上的任意文件(没有系统权限|用户授权)
 2. 不同的标签页/窗口之间无法相互通信(存在同源策略|跨域问题)
 3. 接收网站/域的服务器的信息能力被削弱(请求中 Header 需要指定通信协议)

# JavaScript 的与众不同
 1. 与 HTML/CSS 完全集成。
 2. 主流浏览器支持 JavaScript 脚本。
 3. 弱类型语言,操作简单。
```
